Scikit-Learn Pipeline

Our backend uses a pre-trained machine learning model (best_house_price_model.pkl) serialized via Pickle. The model is structured as a robust pipeline that automatically handles data transformations before making predictions.

1

LabelEncoder (Location)

Applies LabelEncoder to the categorical Location feature, converting 254 unique Bangalore location names into a mathematical matrix the model can interpret. Rare locations (fewer than 10 occurrences) are grouped as "other".

2

StandardScaler (Scaling)

Normalizes all 15 numerical features so they have a mean of 0 and standard deviation of 1. This ensures large numbers like area (sq ft) don't dominate smaller numbers like bedroom counts during prediction.

3

{{ model_name }} (Estimator)

The core algorithm trained on 10,580 Bangalore property records. It learns complex non-linear relationships between property features and prices to predict the final house price in Lakhs (₹).

4

Feature Engineering (Derived)

Additional computed features: total_rooms, area_per_bedroom, amenities_count, has_multiple_amenities, high_amenity_property are automatically derived from user inputs.
